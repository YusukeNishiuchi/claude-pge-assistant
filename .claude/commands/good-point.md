直近の成果物について、良かった点・今後の採点基準にしたい点をヒアリングし、
`criteria/evaluator-criteria.md` に追記します。

## 実行手順

### Step 1: フィードバックをヒアリングする

ユーザーに以下を質問してください:

```
直近の成果物について教えてください。

Q1. 良かった点・気に入った点は何ですか？
Q2. 今後の成果物にも同様の基準を適用したい点はありますか？

（任意）Q3. 逆に改善してほしい点があれば教えてください。
```

---

### Step 2: 採点基準ファイルを更新する

`criteria/evaluator-criteria.md` を読み込み、
「蓄積された基準」セクションに以下の形式で追記してください:

```markdown
### YYYY-MM-DD 追加
- （ユーザーのフィードバックを簡潔に箇条書きで記載）
- （複数ある場合は行を分ける）
```

ユーザーから否定的なフィードバック（Q3）があった場合は、以下のセクションに追記してください:

```markdown
### YYYY-MM-DD 禁止・注意事項
- （避けるべき表現・パターンを具体的に記載）
```

---

### Step 3: 完了報告

以下の形式で報告してください:

```
✅ 採点基準を更新しました

[更新内容]
（追記した内容を1〜2行で要約）

次回の /do-task から反映されます。
```
