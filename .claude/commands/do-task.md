ユーザーから受け取ったタスク依頼を、Planner→Generator→Evaluatorの三層構造で処理します。

## 実行手順

### Step 1: タスク仕様書の作成（Planner）

plannerエージェントに以下を渡してタスク仕様書を作成させてください:

```
以下の依頼をタスク仕様書に変換してください。

【依頼内容】
$ARGUMENTS
```

仕様書が完成したら、Step 2に進みます。

---

### Step 2: 成果物の作成（Generator）

generatorエージェントに以下を渡して成果物を作成させてください:

```
以下のタスク仕様書に基づいて成果物を作成し、outputs/ に保存してください。

【タスク仕様書】
（Step 1で作成した仕様書をそのまま貼り付ける）
```

成果物が保存されたら、Step 3に進みます。

---

### Step 3: 品質評価（Evaluator）＋ 改善ループ

evaluatorエージェントに以下を渡して採点させてください:

```
以下のタスク仕様書と成果物を評価してください。
criteria/evaluator-criteria.md も参照してください。

【タスク仕様書】
（Step 1の仕様書）

【成果物】
（Step 2で作成した成果物の内容）
```

#### 合格（90点以上）の場合
→ Step 4へ進む

#### 不合格（89点以下）の場合
→ generatorエージェントに以下を渡して改善させてください:

```
Evaluatorから以下の改善指示があります。成果物を修正してください。

【改善指示】
（Evaluatorの「Generatorへの指示」をそのまま貼り付ける）

【修正対象ファイル】
（成果物のファイルパス）
```

改善後、再度Step 3の採点を行います。
**最大3回まで** このループを繰り返します。

#### 3回ループしても不合格の場合
以下のメッセージをユーザーに表示してください:

```
3回の改善を試みましたが、90点に達しませんでした。
現在の最新スコア: XX点

選択肢:
1. 依頼内容を変更して再度 /do-task を実行する
2. このまま成果物を使用する（outputs/ に保存済み）
```

---

### Step 4: 完了報告

以下の形式で報告してください:

```
✅ タスク完了

[成果物] outputs/YYYYMMDD_{タスク名}.md
[スコア] XX点

良かった点があれば /good-point で教えてください。
次回からその基準をEvaluatorが学習します。
```
