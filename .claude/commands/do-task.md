ユーザーから受け取ったタスク依頼を、Planner→Generator→Evaluatorの三層構造で処理します。

## 実行手順

### Step 1a: 論点整理（Planner - clarifyモード）

plannerエージェントに以下を渡して論点整理を行わせてください:

```
以下の依頼について論点整理を行ってください。モード: clarify

【依頼内容】
$ARGUMENTS
```

#### 確認が必要な論点がある場合（判定が「ユーザー確認が必要」）

論点整理シートの「確認が必要な論点」をユーザーに提示し、回答を求めてください:

```
以下の点を確認させてください。

（論点整理シートの「確認が必要な論点」を転記）

回答をお願いします。
```

ユーザーの回答を受け取ったら、Step 1b に進みます。

#### 確認不要の場合（判定が「確認不要」）

そのまま Step 1b に進みます。

---

### Step 1b: タスク仕様書の作成（Planner - specモード）

plannerエージェントに以下を渡してタスク仕様書を作成させてください:

```
以下の依頼と確認結果をもとにタスク仕様書を作成してください。モード: spec

【依頼内容】
$ARGUMENTS

【論点整理の結果】
（Step 1aの論点整理シートをそのまま貼り付ける）

【ユーザー確認結果】
（確認があった場合はその回答。なければ「確認不要」）
```

仕様書が完成したら、Step 2に進みます。

---

### Step 2: 成果物の作成（Generator）

generatorエージェントに以下を渡して成果物を作成させてください:

```
以下のタスク仕様書に基づいて成果物を作成し、outputs/ に保存してください。

【タスク仕様書】
（Step 1で作成した仕様書をそのまま貼り付ける）
```

成果物が保存されたら、Step 3に進みます。

---

### Step 3: 品質評価（Evaluator）＋ 改善ループ

evaluatorエージェントに以下を渡して採点させてください:

```
以下のタスク仕様書と成果物を評価してください。
criteria/evaluator-criteria.md も参照してください。

【タスク仕様書】
（Step 1の仕様書）

【成果物】
（Step 2で作成した成果物の内容）
```

合否の判定基準は `rules/scoring-rules.md` の合格ラインを参照してください。

#### 合格の場合
→ Step 4へ進む

#### 不合格の場合
→ generatorエージェントに以下を渡して改善させてください:

```
Evaluatorから以下の改善提案があります。成果物を修正してください。

【改善提案】
（Evaluatorの「改善提案」をそのまま貼り付ける）

【修正対象ファイル】
（成果物のファイルパス）
```

改善後、再度Step 3の採点を行います。
**最大3回まで** このループを繰り返します。

#### 3回ループしても不合格の場合
以下のメッセージをユーザーに表示してください:

```
3回の改善を試みましたが、合格ラインに達しませんでした。
現在の最新スコア: XX点

選択肢:
1. 依頼内容を変更して再度 /do-task を実行する
2. このまま成果物を使用する（outputs/ に保存済み）
```

---

### Step 4: 完了報告

以下の形式で報告してください:

```
✅ タスク完了

[成果物] （Step 2でgeneratorが報告したファイルパス）
[スコア] XX点

良かった点があれば /good-point で教えてください。
次回からその基準をEvaluatorが学習します。
```
